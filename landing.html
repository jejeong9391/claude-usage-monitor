How it was built

핵심 난제: 공식 수치를 어디서 가져오나

로컬 로그 기반 ccusage는 한도 %와 재설정 시각이 실제 서버 집계와 어긋난다. 해결의 출발점은 “Claude Code의 /usage는 대체 어디를 호출하는가”를 찾는 것이었다.

1 엔드포인트 역추적

Claude Code 바이너리의 문자열을 덤프해 OAuth 경로를 탐색.

# 번들 바이너리에서 endpoint 추출
strings claude | grep 'oauth'
# → /api/oauth/usage  ✦ 발견
2 인증 방식 확인

Keychain에 저장된 OAuth 액세스 토큰을 Bearer로 사용.

curl https://api.anthropic.com/api/oauth/usage \
  -H "Authorization: Bearer $TOKEN" \
  -H "anthropic-beta: oauth-2025-04-20"
3 응답 검증

utilization은 0–100(%), resets_at은 실제 서버 윈도우. /usage 화면과 일치 확인.

// 실측 응답
{ "five_hour": {
    "utilization": 58.0,
    "resets_at": "...T14:00:00Z" } }
Architecture

두 데이터 소스를 합치는 하이브리드

정확함은 공식 API에서, 풍부한 상세는 로컬 ccusage에서. 두 Provider는 독립적이라 한쪽이 실패해도 나머지는 계속 표시된다(graceful degradation).

OfficialUsageProvider

GET /api/oauth/usage

  • 정확한 % · 재설정 시각
  • Keychain OAuth 토큰
  • 5h · 주간 · Sonnet · Opus
CCUsageProvider

Process → ccusage blocks/daily

  • 비용 · Burn rate · 예측
  • 토큰 세부 · 모델
  • ~/.claude 로컬 로그
UsageStore (ObservableObject) — 60초 폴링으로 병합 → NSStatusItem + SwiftUI 드롭다운
Code structure

컴포넌트별 단일 책임 분리

6개의 Swift 파일로, 각 파일이 하나의 책임만 갖도록 구성. swiftc가 함께 컴파일한다.

claude-usage-monitor/
├─ src/
│  ├─ Models.swift     Codable + 파싱
│  ├─ Providers.swift  3개 데이터원
│  ├─ Store.swift      병합/게시
│  ├─ Theme.swift      토큰+공통뷰
│  ├─ Views.swift      드롭다운 UI
│  └─ main.swift       진입점
├─ Info.plist        LSUIElement
└─ docs/specs/...     설계 문서

Implementation notes

눈여겨볼 구현 기법

Xcode·SPM 없이 단일 바이너리로 떨어지도록, 의존성을 최소화한 선택들.

Keychain을 CLI로 우회

Security 프레임워크 대신 /usr/bin/security를 Process로 호출 — ccusage 호출과 동일 패턴이라 코드가 단순하고, 권한 프롬프트도 회피된다.

let task = Process()
task.executableURL = .init(fileURLWithPath: "/usr/bin/security")
task.arguments = ["find-generic-password",
  "-s", "Claude Code-credentials", "-w"]
// → JSON 의 claudeAiOauth.accessToken 파싱
detached Task + 동기 fetch

백그라운드 Task에서 세마포어로 URLSession을 동기화해 호출하고, 결과만 @MainActor로 넘겨 UI 갱신. 메인 스레드를 막지 않는다.

Task.detached {
  let off = OfficialUsageProvider.fetch()   // 동기
  let blk = CCUsageProvider.activeBlock()
  await self?.apply(off, blk)        // MainActor
}
마이크로초 ISO8601 파싱

응답의 resets_at은 6자리 분수초를 포함한다. ISO8601DateFormatter가 깨지는 케이스라 분수초를 먼저 제거 후 파싱.

// "...:00.976658+00:00" → 분수초 제거
let s = raw.replacingOccurrences(
  of: #"\.\d+"#, with: "",
  options: .regularExpression)
부분 실패 허용

공식 호출 결과를 ok / noToken / unauthorized / offline로 구분. 마지막 성공값은 유지하고 상태만 표시해, 오프라인에도 직전 값을 보여준다.

enum OfficialResult {
  case ok(OfficialUsage)
  case noToken, unauthorized, offline
}
Build & run

Xcode 없이 한 줄로 빌드

SwiftPM·프로젝트 파일 없이 swiftc로 직접 컴파일한 뒤 .app 번들로 감싸고 ad-hoc 서명한다.

# 1) 컴파일
swiftc -O -framework Cocoa src/*.swift \
  -o build/ClaudeUsageMonitor

# 2) .app 번들 구성 + ad-hoc 서명
codesign --force --sign - \
  ~/Applications/ClaudeUsageMonitor.app

# 3) 실행 (메뉴바에 상주)
open ~/Applications/ClaudeUsageMonitor.app

알아둘 점

/api/oauth/usage는 비공개 내부 엔드포인트다. 응답 형태가 바뀌면 공식 카드는 “오프라인”으로 전환되고 ccusage 상세만 유지된다. 본인 토큰·읽기 전용·개인 모니터링 범위.

이 프로젝트가 보여주는 것

작은 도구지만, 실무에서 쓰이는 기법들이 압축돼 있다.

바이너리 리버싱으로 엔드포인트 발견 OAuth 토큰 + Keychain 연동 이종 데이터 소스 병합 graceful degradation 설계 Swift 동시성 (Task/@MainActor) AppKit ↔ SwiftUI 브릿지 swiftc-only 빌드 파이프라인 단일 책임 모듈 분리
